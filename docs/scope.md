# 项目范围

## 目标场景

该平台面向一次校内 CTF 招新比赛，允许校外访问。

主要服务对象有三类：

- 选手：注册、参赛、看题、启动实例、提交 Flag、查看排行
- 管理员：管理题目、公告、用户、提交记录、实例和审计日志
- 运维或出题人：维护题目资源、运行环境、附件和服务健康状态

## 已确认定位

当前项目按“单场比赛平台”推进，不在当前阶段扩展为长期训练社区或多比赛运营后台。

已确认方向：

- 个人赛
- 单场比赛模型优先
- 动态容器实例纳入首期能力
- 单机部署优先，但面向 `500 ~ 1000` 注册量做生产化加固

## 首期目标能力

### 账号与权限

- 注册
- 登录
- Bearer Token 鉴权
- 用户角色区分
- 基础用户资料与状态管理

### 比赛与题目

- 公告展示
- 题目分类、分值、难度、排序
- 题面内容管理
- 附件上传与下载
- 题目隐藏与开放控制
- 动态题运行配置

### 提交与排行

- Flag 提交
- 正误反馈
- 解题记录
- 个人提交记录
- 排行榜

### 管理后台

- 用户列表与状态调整
- 题目增改查
- 公告发布与删除
- 提交记录查询
- 动态实例状态查看与干预
- 管理操作审计视图

### 基础运维

- 健康检查
- 配置化部署
- 实例生命周期清理
- 备份与恢复方案
- 基础日志与指标

## 当前明确不纳入范围的能力

- 复杂团队模式
- 站内讨论区
- 多比赛并发运营平台
- 多租户
- GeekGame 式复杂分布式状态机
- Kubernetes 调度
- 大型公开赛级反作弊系统
