# 项目范围

## 目标场景

该平台面向一次 CTF 招新流程，主要服务对象有三类：

- 选手：注册、参赛、查看题目、启动实例、提交 Flag、查看排行
- 管理员：配置比赛、管理题目、查看提交记录、管理公告与实例状态
- 运维或出题人：维护题目资源、容器实例、附件和服务健康状态

## 已确认定位

当前项目按 `单场 CTF 招新比赛平台` 推进，而不是一开始就做长期训练社区。

同时，以下方向已经确认：

- 个人赛
- 单场比赛模型优先
- 动态容器实例首期纳入 MVP
- 单机 Docker Compose 部署优先

## 首期必须做的能力

### 账号与权限

- 注册
- 登录
- JWT 鉴权
- 用户角色区分
- 基础个人资料

### 比赛与题目

- 比赛基本信息配置
- 题目分类
- 题目分值
- 题面内容管理
- 附件上传与下载
- 题目隐藏与开放控制
- 动态实例题目的运行配置

### 提交与排行

- Flag 提交
- 正误反馈
- 首血记录
- 解题时间记录
- 排行榜

### 管理后台

- 用户列表
- 题目增删改查
- 提交记录查询
- 公告发布
- 比赛启停控制
- 动态实例状态查看

### 基础运维

- 日志
- 健康检查
- 配置化部署
- 备份方案
- 实例生命周期清理

## 首期可选能力

- 动态实例续期次数限制
- 多附件题目
- 邮件通知
- 管理操作审计视图
- 公告置顶
- Redis 限流

## 明确不纳入首期的能力

- 复杂团队模式
- 站内讨论区
- Writeup 平台
- 多比赛并发运营后台
- 多租户
- Kubernetes 调度
- 复杂风控系统
