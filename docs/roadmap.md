# 开发路线

## 阶段 0：准备阶段

目标：固定范围、技术和开发方向。

状态：已完成首轮文档基线。

## 阶段 1：工程初始化

目标：搭出可运行的基础工程，并把动态实例相关结构提前埋好。

交付物：

- 前后端脚手架
- Docker Compose 开发环境
- PostgreSQL 初始迁移
- 基础健康检查接口
- 动态实例管理接口占位
- 后台 sweeper 入口

## 阶段 2：基础业务闭环

目标：先打通登录、题目浏览和动态实例生命周期。

交付物：

- 注册登录
- 用户信息接口
- 题目列表与详情
- 动态实例启动、查看、销毁
- 题目运行配置管理
- 管理端题目增删改查

## 阶段 3：比赛核心闭环

目标：完成招新赛的核心业务。

交付物：

- Flag 提交
- 解题记录
- 排行榜
- 公告管理
- 比赛启停控制
- 提交记录查询

## 阶段 4：加固与上线准备

目标：补齐稳定性和运维能力。

交付物：

- 权限补强
- 审计日志
- 基础限流
- 实例异常恢复与清理补强
- 关键流程测试
- 部署文档

## 当前建议优先级

1. 完成数据库表结构和迁移执行方式
2. 完成认证与用户模型
3. 接入真实 Docker 实例管理器
4. 完成题目与实例管理后台
5. 打通提交、积分和排行
