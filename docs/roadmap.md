# 开发路线

## 阶段 0：准备阶段

目标：固定范围、技术和开发方向。

状态：已完成首轮文档基线。

## 阶段 1：工程初始化

目标：搭出可运行的基础工程，并把动态实例相关结构提前埋好。

状态：已完成。

交付物：

- 前后端脚手架
- Docker Compose 开发环境
- PostgreSQL 初始迁移
- 基础健康检查接口
- 动态实例管理接口
- 后台 sweeper 入口

## 阶段 2：基础业务闭环

目标：打通登录、看题、动态实例和 Flag 提交主链路。

当前状态：进行中，已完成以下部分：

- 注册登录
- `GET /me`
- 题目列表与题目详情
- 动态实例启动、查看、销毁
- 公告列表
- Flag 提交
- 排行榜查询

仍待补齐：

- 用户提交历史
- 管理端题目和公告管理
- 提交记录后台查询

## 阶段 3：比赛核心闭环

目标：完成招新赛的完整业务。

交付物：

- 题目管理后台
- 公告管理后台
- 管理端实例监控与强制终止
- 提交记录查询
- 权限细化

## 阶段 4：加固与上线准备

目标：补齐稳定性和运维能力。

交付物：

- 权限补强
- 审计日志
- 基础限流
- 实例异常恢复与清理补强
- 关键流程测试
- 部署文档

## 当前建议优先级

1. 完成管理端题目、公告和提交记录 API
2. 完成前端登录流、题目详情页和实例面板
3. 补充实例恢复与管理员强制终止
4. 再进入管理员后台页面开发
