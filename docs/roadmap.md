# 开发路线

## 阶段 0：准备阶段

目标：固定范围、技术和开发方向。

状态：已完成首轮文档基线。

## 阶段 1：工程初始化

目标：搭出可运行的基础工程，并把动态实例相关结构提前埋好。

状态：已完成。

交付物：

- 前后端脚手架
- Docker Compose 开发环境
- PostgreSQL 初始迁移
- 基础健康检查接口
- 动态实例管理接口
- 后台 sweeper 入口

## 阶段 2：基础业务闭环

目标：打通登录、看题、动态实例和 Flag 提交主链路。

状态：已完成基础版本。

已完成：

- 注册登录
- `GET /me`
- 题目列表与题目详情
- 动态实例启动、查看、销毁
- 公告列表
- Flag 提交
- 排行榜查询

## 阶段 3：管理端闭环

目标：补齐出题人和管理员的运营能力。

状态：进行中。

已完成：

- 管理员 RBAC 中间件
- 管理端题目列表、创建、更新
- 管理端公告列表、创建
- 管理端提交记录列表
- 管理端实例列表与强制终止

仍待补齐：

- 题目附件上传
- 运行配置管理接口
- 用户管理
- 审计日志接口

## 阶段 4：加固与上线准备

目标：补齐稳定性和运维能力。

交付物：

- 权限补强
- 审计日志
- 基础限流
- 实例异常恢复与清理补强
- 关键流程测试
- 部署文档

## 当前建议优先级

1. 完成题目运行配置与附件管理 API
2. 完成用户管理与管理员权限细化
3. 完成前端登录流、题目详情页和实例面板
4. 再进入管理员后台页面开发
